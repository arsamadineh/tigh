# سهم‌گذاری در تیغ

از علاقه‌مندی شما برای سهم‌گذاری در تیغ متشکریم! این سند شامل راهنمایی‌ها و دستورالعمل‌های سهم‌گذاری است.

## آداب و رسوم

در تمام تعاملات احترام‌گذار و سازنده باشید. ما سهم‌گذاران از تمام اصول را خوش‌آمد می‌گویم.

## چگونه سهم‌گذاری کنم

### گزارش باگ

1. بررسی کنید که باگ قبلاً گزارش نشده است
2. توضیح واضح از مشکل ارائه دهید
3. مراحل تکرار مشکل را شامل کنید
4. رفتار مورد انتظار در مقابل واقعی را توضیح دهید
5. محیط خود را شامل کنید (OS، نسخه Node، نسخه TypeScript)

### پیشنهاد بهبودی

1. بررسی کنید که پیشنهاد قبلاً داده نشده است
2. یک مورد استفاده واضح ارائه دهید
3. توضیح دهید چرا این ویژگی مفید است
4. مثال‌ها ارائه دهید

### ارسال تغییرات کد

1. Repository را Fork کنید
2. شاخه جدید ایجاد کنید: `git checkout -b feature/نام-ویژگی`
3. تغییرات را انجام دهید
4. تست بنویسید یا به‌روزرسانی کنید
5. بررسی کنید کد TypeScript از آزمایش عبور کند: `npm run typecheck`
6. Commit کنید: `git commit -m 'Add feature: توضیح'`
7. Push کنید: `git push origin feature/نام-ویژگی`
8. Pull Request را با توضیح واضح باز کنید

## تنظیم توسعه

```bash
# کلون کردن Repository
git clone https://github.com/arsamadineh/tigh.git
cd tigh

# نصب وابستگی‌ها
npm install

# ساخت پروژه
npm run build

# بررسی انواع
npm run typecheck
```

## سبک کد

- از TypeScript برای تمام کد استفاده کنید
- سبک کد موجود را دنبال کنید
- از نام‌های معنی‌دار استفاده کنید
- برای منطق پیچیده نظر اضافه کنید
- توابع را کوچک و متمرکز نگه دارید

## راهنمایی Commit

- از حالت امری استفاده کنید ("Add feature" نه "Added feature")
- از یک فعل شروع کنید
- خط اول را کمتر از 70 کاراکتر نگه دارید
- موارد را هنگام مرتبط بودن ذکر کنید: "Fix #123"

## فرایند Pull Request

1. مستندات را بر‌حسب نیاز به‌روزرسانی کنید
2. تست بیفزایید یا به‌روزرسانی کنید
3. بررسی کنید تمام آزمایش‌ها عبور کنند
4. توضیح واضح PR ارائه دهید
5. مسائل مرتبط را لینک کنید

## مجوز

با سهم‌گذاری، موافقت می‌کنید که سهم‌گذاری‌های شما تحت مجوز MIT مجاز است.
